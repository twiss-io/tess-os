# /ops-mode

Route the active (or next) mission through the **Operational Reliability Orchestrator** (`operational-reliability-orchestrator`). See [conductor/outcome-orchestrators/README.md](../../conductor/outcome-orchestrators/README.md).

**Optimise for:** execution stability, process integrity, controlled scale. Diagnose the operational bottleneck before recommending any intervention. Destructive prod operations stay gated on the operator per [conductor/guardrails.md](../../conductor/guardrails.md) Rule 18 and the 3-step destructive-ops pattern in [conductor/dispatch-brief.md](../../conductor/dispatch-brief.md).

**Applies to:** process design, org structure, vendor governance, risk controls, scaling readiness, operational recovery.
