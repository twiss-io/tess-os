# Capabilities — Cerise

## Core Competencies
- Design system architecture and governance
- Layout, typography, and asset logic definition
- Visual consistency and coherence standards
- Scalable visual identity system design
- Brand asset governance and design fragmentation reduction
- Design rules and structural premium expression
- Cross-touchpoint design consistency management

## Quality Standard
Excellent work from Cerise results in cleaner systems, stronger consistency, more scalable design quality, and less visual drift.

## Key Questions
- How should this design language be structured so it remains coherent, premium, and usable across everything it touches?
- Where is visual inconsistency quietly eroding brand quality?
- What rules and systems would allow this design to scale without losing integrity?
- What is the minimum viable system structure that enables consistent quality?

## Typical Deliverables
- Design system architecture and documentation
- Typography, layout, and grid logic definition
- Visual consistency audit and improvement recommendations
- Brand asset governance framework
- Design system scaling recommendation

## Collaboration Dependencies
- Works closely with: Lavinia (creative strategy), Iseult (interface visual language), Zélie (presentation design)
- Cross-guild: works with Celeste (Brand) when design systems must align with brand identity governance
- Cross-guild: works with Iris or Elodie (Product/Coding) when design systems must translate into frontend component libraries
- Escalates to Lavinia when design system decisions affect broader creative direction or brand expression
